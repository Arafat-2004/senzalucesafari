# Phase 5 — Data Fetching Optimization
Read every public-facing page file fully. Identify the data fetching pattern used on each page.
## Add caching to all Prisma queries on public pages
For every Prisma query on a public-facing page (not admin pages), wrap it with Next.js unstable_cache:
```typescript
import { unstable_cache } from 'next/cache'
const getCachedTours = unstable_cache(
  async () => {
    return await prisma.tour.findMany({ where: { status: 'published' } })
  },
  ['tours-list'],
  {
    revalidate: 3600, // revalidate every 1 hour
    tags: ['tours'],
  }
)
```
Apply appropriate cache durations:
- Tour/package listings: revalidate every 3600 seconds (1 hour)
- Destination listings: revalidate every 3600 seconds
- Blog post listings: revalidate every 1800 seconds (30 minutes)
- Homepage featured content: revalidate every 3600 seconds
- Individual tour/destination/blog detail pages: revalidate every 7200 seconds (2 hours)
Do NOT cache admin pages — admin data must always be fresh.
Do NOT cache booking-related queries — bookings must always be real-time.
## Add generateStaticParams for dynamic routes
Find every dynamic route in the project: `[slug]`, `[id]`, `[category]` etc. in `src/app/`.
For each dynamic route page that is public-facing, add `generateStaticParams`:
```typescript
export async function generateStaticParams() {
  const tours = await prisma.tour.findMany({
    where: { status: 'published' },
    select: { slug: true },
  })
  return tours.map((tour) => ({ slug: tour.slug }))
}
```
Also add to each dynamic page:
```typescript
export const dynamicParams = true // allow new slugs not in generateStaticParams
export const revalidate = 3600 // ISR — regenerate page every 1 hour
```
## Parallel data fetching
Find any page that makes multiple sequential database calls like:
```typescript
const tours = await prisma.tour.findMany(...)
const destinations = await prisma.destination.findMany(...)
const featured = await prisma.featured.findFirst(...)
```
Replace all sequential awaits with parallel fetching using Promise.all:
```typescript
const [tours, destinations, featured] = await Promise.all([
  prisma.tour.findMany(...),
  prisma.destination.findMany(...),
  prisma.featured.findFirst(...),
])
```
This can cut page data loading time by 50-70% on pages with multiple queries.
